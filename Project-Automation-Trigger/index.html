The High Cost of Reactive Inventory

Traditional supply chains rely on manual reports and delayed metrics. By the time a critical SKU is flagged, the stockout has already occurred, halting operations and costing money.

The Solution: Proactive Webhook Alerting

This system continuously monitors daily consumption rates against dynamic lead times. The moment current stock dips below the mathematical Reorder Point (ROP), an automated payload is fired directly to the procurement team—days before the bin goes empty.

Under the Hood: How It Works

Dynamic Data Simulation

A Python-driven engine generating live, fluctuating consumption metrics across 50+ critical SKUs to stress-test the alerting logic.

Real-Time Webhooks

Node.js listeners calculate dynamic safety stocks and trigger instant JSON payloads the millisecond an item hits its custom Reorder Point.

Power BI Integration

Live streaming datasets translated through advanced DAX formulas to provide a bird's-eye view of 'Total Value at Risk' and 'Days Until Stockout'.

# Automated Webhook Trigger Logic
import requests
import json

def check_stock_levels(sku_data):
    for item in sku_data:
        if item['current_stock'] <= item['reorder_point']:
            payload = {
                "event": "stockout_alert",
                "sku": item['sku'],
                "current_stock": item['current_stock'],
                "reorder_point": item['reorder_point'],
                "lead_time": f"{item['lead_time_days']} days",
                "status": "CRITICAL"
            }
            # Fire webhook to procurement channel
            response = requests.post(WEBHOOK_URL, json=payload)
            print(f"Alert sent for {item['sku']}: {response.status_code}")
